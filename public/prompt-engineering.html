Complex Project #1

Knowledge Graph-Based Personalized Retrieval

Dynamically generating context-aware prompts from user knowledge graphs across millions of records

The Challenge

Generic prompts fail to leverage individual user context—preferences, interaction history, and behavioral patterns. I needed to build a system that dynamically generates personalized prompts at scale without sacrificing latency.

Prompt Engineering Approach

  • ▸
    Knowledge Graph Construction: Extracted entities, relationships, and learning patterns from millions of user records to build individual knowledge graphs for each user
  • ▸
    Dynamic Context Injection: Engineered prompts that dynamically incorporate user graph features—expertise level, topic preferences, interaction patterns—to tailor retrieval results
  • ▸
    Template Architecture: Built modular prompt templates with variable context injection points, allowing personalization without rewriting entire prompts
  • ▸
    Balancing Context vs. Latency: Optimized graph traversal and context selection to include maximum relevant information while keeping prompt token count under 2000 for sub-500ms generation

Systematic Iteration Process

Testing Methodology:

  • • Built evaluation dataset of 1000+ user queries
  • • Measured retrieval relevance with human annotations
  • • A/B tested prompt variants across user cohorts
  • • Tracked personalization impact on engagement

Quality Metrics:

  • • Semantic similarity to expected results
  • • User satisfaction scores (implicit/explicit)
  • • Refinement rates (users rephrasing queries)
  • • Cold-start performance improvements

Complex Project #2 · Current Work at Nurture

Deterministic Multimodal Pipeline with Structured Output

Achieving reliable JSON generation from multimodal inputs for spatial reasoning and interactive gaming

The Challenge

Converting natural language user intent (e.g., "move this object to the left") combined with visual context into structured JSON actions that game systems can execute reliably. The core challenge: achieving deterministic behavior from non-deterministic LLMs.

Example Input:

"Move the character to the left corner near the tree"

Required Output (JSON):

{ "action": "move", "direction": "left", "target": "tree", "distance": 2.5 }

Prompt Engineering Techniques

  • ▸
    Schema-Constrained Prompts: Designed prompts with explicit JSON schema definitions, valid action primitives, and constraint boundaries (e.g., direction must be: left, right, forward, back)
  • ▸
    Few-Shot Examples: Included 5-10 example input→output pairs in prompts, covering edge cases (ambiguous directions, multiple objects, relative positioning)
  • ▸
    Multimodal Context Integration: Engineered prompts that process both text instructions AND visual scene context (object positions, spatial relationships) to generate spatially-aware actions
  • ▸
    Iterative Refinement Loops: Built validation layers that catch invalid outputs and re-prompt with error context (e.g., "Invalid direction 'leftward' - must be one of: left, right, forward, back")
  • ▸
    Task Decomposition: Split complex commands into sub-tasks (intent classification → entity extraction → action generation → validation) rather than one-shot generation

Quality Assurance & Iteration

Automated Validation Gates:

  • • JSON schema validation (required keys, value types, range constraints)
  • • Spatial consistency checks (actions valid given current scene state)
  • • Error handling with automatic retry + error context injection
  • • Timeout limits with fallback to simpler prompts

Systematic Testing:

  • • 200+ test cases covering edge cases, ambiguous inputs, multi-object scenarios
  • • Regression testing on every prompt iteration
  • • Cross-model evaluation (GPT-4, Claude, Gemini) for consistency
  • • Performance benchmarking (latency, token efficiency, success rates)

OpenAI API Claude API LangChain Pydantic JSON Schema FastAPI Python

Achieved 95%+ valid JSON output rate in production with sub-200ms latency, enabling real-time interactive gaming experiences for 6K+ active sessions. This required 40+ prompt iterations, comprehensive test coverage, and systematic quality gates.

Case Study: Educational AI Quality Analysis

As part of my research into educational AI systems, I conducted a deep analysis of Brainscape's AI Flashcard Copilot—an 8-prompt system for improving flashcard quality. This demonstrates my approach to evaluating prompt quality and identifying improvement opportunities.

Identified Gaps in Current Implementation

  • ✗
    Missing Output Constraints: Documentation reveals "ChatGPT will not adhere to the number of flashcards requested" and "does strange things converting to new languages" → suggests missing validation and schema enforcement
  • ✗
    Hallucination Risk: "Answer this card" prompt generates answers even with empty answer fields → lacks grounding in source material or deck context
  • ✗
    Card-Level vs. Deck-Level Understanding: AI operates on individual cards without understanding knowledge graph relationships across the entire deck
  • ✗
    Generic Model Usage: Uses general-purpose ChatGPT rather than education-specialized or fine-tuned models

Proposed Improvements (My Approach)

  • ✓
    Deck-Level Knowledge Graphs: Build graphs capturing concept relationships across cards to ensure consistency, detect gaps, and suggest optimal card sequences
  • ✓
    Validation Layers: Implement automated checks (answer leakage detection, hint relevance scoring, mnemonic-to-answer mapping validation) before showing outputs to users
  • ✓
    Confidence Scoring: Add confidence metrics to enable auto-accept for high-quality outputs and human review for uncertain generations
  • ✓
    Evaluation Framework: Build test suites for each prompt verifying outputs match pedagogical best practices (e.g., "Simplify" actually reduces word count, "Refine card" preserves semantic meaning)

My Prompt Engineering Methodology

1. Discovery & Requirements

Define success criteria, edge cases, and quality metrics before writing a single prompt

• What constitutes "good" output? (quantifiable metrics)

• What are unacceptable failures? (hallucinations, invalid formats, toxicity)

• What context is available? (user data, source documents, system state)

2. Prompt Architecture Design

Structure prompts with clear instructions, constraints, examples, and output formats

• Role & context setting

• Explicit constraints and boundaries

• Few-shot examples (positive & negative)

• Output schema with validation rules

3. Systematic Testing & Iteration

Build comprehensive test suites and iterate based on failure analysis

• Create 100+ test cases covering happy paths and edge cases

• Version control prompts (treat like code)

• Change one variable at a time, measure impact

• Document why each change was made

4. Production Validation

Deploy with automated quality gates and continuous monitoring

• Automated validation layers (schema, content, safety)

• Confidence-based routing (auto-ship vs. human review)

• Real-time monitoring dashboards

• Weekly manual spot checks of random samples

5. Continuous Improvement

Use production feedback to identify systematic issues and improve prompts

• Track acceptance rates, edit patterns, regeneration frequency

• Failed outputs become regression test cases

• A/B test prompt improvements before full rollout

• Maintain prompt performance benchmarks over time

Key Learnings

1. Structured Outputs Require Structured Prompts

Moving from free-form responses to constrained JSON outputs requires explicit schemas, clear validation rules, and comprehensive examples covering edge cases.

2. Context Engineering Beats Prompt Tweaking

The most powerful prompts aren't static—they dynamically incorporate user context, domain knowledge, and system state. Investing in context infrastructure yields better results than endlessly tweaking prompt wording.

3. Automated Testing Is Non-Negotiable

You can't rely on user feedback as your primary quality signal. Build test suites, regression tests, and automated validation gates that catch issues before users see them.

4. Decomposition Improves Reliability

Breaking complex tasks into smaller, verifiable steps produces more reliable outputs than one-shot generation. Each step can be validated independently.

5. Production Prompts Are Code

Treat prompts like production code: version control, documentation, testing, code review, and careful deployment. Track performance metrics and maintain benchmarks over time.
